# Advanced Form System - Implementation Plan

## Overview
Build 10 reusable Blazor Server form components following the existing `fc-` BEM design system, with JS interop via `window.FCForms` namespace.

## Files to Create

### Components (in `Components/Shared/Forms/`)
1. **FormField.razor** - Wrapper with label, hint, validation states (blur validation, error slide-in, success checkmark)
2. **FormSection.razor** - Collapsible accordion sections for long forms
3. **FormActions.razor** - Standardized form footer (primary/secondary/danger actions + auto-save indicator)
4. **CurrencyInput.razor** - Auto-formatting with thousand separators, currency prefix, decimal binding
5. **DateRangePicker.razor** - Dropdown calendar with single/range modes + presets (Today, This Week, etc.)
6. **MultiSelect.razor** - Searchable dropdown with pill tags, keyboard nav, max selections
7. **FileUpload.razor** - Drag-and-drop zone with progress, file type/size validation, image preview
8. **RichTextEditor.razor** - contenteditable-based with Bold/Italic/Underline/Lists/Links toolbar
9. **AutoSaveIndicator.razor** - Saving.../Saved/Unsaved states badge
10. **ConditionalField.razor** - Animated show/hide with smooth height transitions

### Supporting Files
- **wwwroot/js/forms.js** - JS interop (`window.FCForms`) for currency formatting, outside-click, drag-drop, rich text commands, calendar positioning
- **wwwroot/css/forms.css** - All form component styles using `fc-` BEM convention + CSS variables

## Implementation Order
1. ConditionalField, AutoSaveIndicator (no dependencies)
2. FormField (core wrapper - used by everything)
3. FormSection, FormActions (layout components)
4. CurrencyInput, DateRangePicker (JS interop inputs)
5. MultiSelect, FileUpload (complex interactive inputs)
6. RichTextEditor (JS interop heavy)
7. JS interop file (forms.js)
8. CSS file (forms.css)

## Component APIs

**FormField**: `Label`, `HintText`, `ErrorMessage`, `IsRequired`, `ValidationState` (None/Valid/Invalid/Validating), `ChildContent`
**FormSection**: `Title`, `Description`, `BadgeText`, `IsOpen`, `ChildContent`
**FormActions**: `PrimaryText`, `SecondaryText`, `OnPrimary`, `OnSecondary`, `IsSaving`, `SaveState`, `ChildContent`
**CurrencyInput**: `@bind-Value` (decimal), `CurrencySymbol` (default "₦"), `Placeholder`, `Disabled`
**DateRangePicker**: `@bind-StartDate`, `@bind-EndDate`, `Mode` (Single/Range), `ShowPresets`
**MultiSelect**: `Items` (list), `@bind-SelectedValues`, `Placeholder`, `MaxSelections`, `DisplayMember`, `ValueMember`
**FileUpload**: `Accept`, `MaxSizeMb`, `Multiple`, `OnFilesSelected`, `ShowPreview`
**RichTextEditor**: `@bind-Value` (HTML string), `MaxLength`, `Placeholder`, `ToolbarOptions`
**AutoSaveIndicator**: `State` enum (Unsaved/Saving/Saved/Error), `LastSavedAt`
**ConditionalField**: `Visible` (bool), `ChildContent`

## Design Conventions
- All CSS uses existing variables: `--cbn-*`, `--space-*`, `--radius-*`, `--shadow-level-*`, `--transition-*`, `--ease-*`
- BEM naming: `.fc-form-field`, `.fc-form-field__label`, `.fc-form-field--valid`, etc.
- Accessibility: ARIA labels, keyboard navigation, focus management, `prefers-reduced-motion`
- JS interop: `window.FCForms.{method}` pattern, disposed via `IAsyncDisposable`
